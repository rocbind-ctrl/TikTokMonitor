# TikTokMonitorReplica 使用说明

## 文件说明

- `TikTokMonitorReplica.exe`：复刻后的 Windows 可执行程序。
- `config.yaml`：程序配置文件，必须和 exe 放在同一目录。
- `data/monitor.db`：SQLite 数据库，保存账号、视频、历史快照、同步日志和告警。
- `data/monitor.db-shm`、`data/monitor.db-wal`：SQLite WAL 辅助文件，运行时可能自动生成。

## 启动方式

双击 `TikTokMonitorReplica.exe` 即可启动。

默认配置为桌面窗口模式：

```yaml
server:
  host: "127.0.0.1"
  port: 8099
  mode: "desktop"
```

如果桌面窗口启动失败，程序会自动退回浏览器模式。

也可以用命令行强制浏览器模式：

```powershell
.\TikTokMonitorReplica.exe --web
```

启动后访问：

```text
http://127.0.0.1:8099
```

如果 8099 被占用，程序会尝试切换到备用端口：`8888`、`9000`、`9100`。

## 登录

当前示例配置启用了 Web 密码：

```text
000000
```

如需修改密码，编辑 `config.yaml`：

```yaml
security:
  web_password: "你的新密码"
```

如果希望关闭登录验证，可将密码留空：

```yaml
security:
  web_password: ""
```

## TikTok 数据源

配置位置：

```yaml
tiktok:
  provider: "auto"
  api_base: "https://tikwm.com/api"
  proxy: ""
```

可选 provider：

- `auto`：推荐，自动组合可用数据源。
- `tikwm`：使用 tikwm API。
- `ytdlp`：使用 yt-dlp 获取视频列表。
- `direct`：直连 TikTok 页面，通常只能获取账号基础信息。

如果当前网络无法访问 TikTok 或 tikwm，可配置代理：

```yaml
tiktok:
  proxy: "http://127.0.0.1:7890"
```

## 数据备份

所有业务数据都在 `data/` 目录。

备份时直接复制整个 `data/` 目录即可。建议在执行大量同步前备份：

```powershell
Copy-Item -Recurse .\data .\data_backup
```

恢复时关闭程序，然后用备份目录替换当前 `data/`。

## 常见问题

### 1. 双击后没有窗口

可尝试命令行启动浏览器模式：

```powershell
.\TikTokMonitorReplica.exe --web
```

如果仍无法访问，请检查防火墙、端口占用或 `config.yaml` 是否在 exe 同级目录。

### 2. 提示端口不可用

修改 `config.yaml` 中的端口：

```yaml
server:
  port: 8888
```

### 3. 同步失败

优先检查：

- 用户名是否正确。
- 当前网络是否能访问 TikTok/tikwm。
- 是否需要在 `config.yaml` 中配置代理。
- 稍后重试，外部接口可能限流或短暂不可用。

### 4. 数据库被占用

确认没有同时打开多个程序实例。关闭所有窗口后再启动。
